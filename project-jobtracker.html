Bottom line

Your inbox already knows where you applied — this app reads it and keeps the tracker for you, from Applied through Offer.

Overview

Job hunting generates a spreadsheet nobody wants to maintain. This full-stack app maintains it for you: sign in with Google or Outlook (read-only access), and Claude reads your application and recruiter emails to extract the company, position, recruiter, and current status — no manual data entry, ever.

Built for my own 2026 job search, and running on it — the demo above is the real app.

How it works
  • Multi-inbox sync — link any number of Gmail and Outlook accounts (Google OAuth 2.0 + Azure AD) with read-only scopes; tokens refresh silently, so sessions never drop.
  • AI classification — Claude extracts structured fields from each email and advances applications through Applied → Screening → Interview → Offer / Rejected as new mail arrives.
  • Built for real inboxes — metadata-only fetching, chunked API requests, and processed-email bookkeeping keep syncs fast and idempotent: old mail is never re-classified, and a failed classification is retried rather than lost.
  • Stays current — auto-refresh every 30 minutes, one-click manual status overrides, CSV export, PostgreSQL persistence (Drizzle ORM), and dark mode.
Results & impact

Every application in one live dashboard with zero upkeep — statuses update themselves as recruiters reply. The engineering work that made it reliable (timeout-safe chunked fetching, idempotent processing, classification-failure recovery) is visible in the commit history ↗.
